# M4-S26 · Diagnóstico técnico-económico de integración IA

## Objetivo

Esta aplicación web estática ayuda a analizar un caso de integración de IA generativa en atención al cliente.

Su objetivo es ordenar el análisis del caso, introducir datos básicos, calcular valores iniciales, comparar alternativas, registrar riesgos y preparar una decisión final justificada.

## Problema que analiza

La aplicación analiza una posible integración de IA generativa en un proceso de atención al cliente.

El problema principal es decidir si conviene utilizar una API externa de IA o una plataforma cloud gestionada, teniendo en cuenta costes, trazabilidad, protección de datos, control técnico, riesgos y escalabilidad.

## Funcionalidades

- Introducción de datos básicos del caso.
- Cálculo de consultas mensuales.
- Cálculo de tokens mensuales de entrada.
- Cálculo de tokens mensuales de salida.
- Estimación de minutos y horas ahorradas.
- Estimación de ahorro humano bruto.
- Matriz comparativa entre API externa de IA y plataforma cloud gestionada.
- Mini-RAID con riesgo, supuesto, incidencia y dependencia.
- Redacción de decisión final.
- Generación de resumen final para copiar en la entrega.
